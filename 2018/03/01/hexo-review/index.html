hexo 블로그 만들고 느낀 점
2018-03-01| 단상|

어째서 블로그를 만들었는가?

저는 컨테이너가 필요했습니다.
무언가를 담을 공간이요.
그래서 GitHub에 이것저것 소스를 담거나,
마크다운에 짧은 글을 섞어서 올리곤 합니다.

하지만 뭔가 소스에 대해 서술하거나,
이미지를 첨부해서 생각을 뻗쳐 나가는 건
github에 막 어울리는 것 같지는 않다고 느꼈어요.

또 개발 외에도 저는 잡다한 생각을 하곤 하는데,
그런 단상들을 적을 공간이 필요하기도 했습니다.
여하튼 그런 평범한 이유로 블로그를 만들기 시작했습니다.

어떤 걸로 만들까?

원래는 AWS EC2와 Django를 이용해서
블로그를 만들려고 했습니다.
근데 그럴려면 해야 되는 게 많더라구요.

“안 해도 되는 일은 안 한다.
해야 한다면 간략하게.”

에너지 절약주의가 문득 머리를 스치며,
GitHub Pages를 이용하는 걸로 우회했습니다.

GitHub + HEXO

한 반년 전쯤인가 연습삼아 GitHub Page를 만든 적이 있었어요.
그 때 사용했던 것이 hexo라서 인연을 이어가기로 결정.
페이지를 올리는 것까진 쉬웠어요.

그리고 테마를 입히기만 하면 되는데…
입히기만 하면 됐는데…

다 중국어더라구요!

여기도 중국어, 저기도 중국어… 고생 깨나 했습니다.
그래도 찾다보니 DOCS를 만들어 놓는 친구들이 많더라구요.
고마운 친구들입니다. Guidance 따라가는 것만으로 블로그를 완성했습니다.

그래서 블로그를 만들고 느낀 점!

나는 코드몽키다!!!

으앙, 내가 코드몽키다. 제가 스스로 뭘 한 게 없어요.
서버는 GitHub가 대주고, 블로그는 HEXO가 올려주고,
테마는 갖다 씌우고, 방문 집계는 Google Analytics,
공유는 AddThis, 덧글은 Disqus
코딩 100줄도 안 한 거 같은데…

여튼 좋은 세상입니다.
